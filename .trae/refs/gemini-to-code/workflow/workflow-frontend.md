## **🎨 视觉驱动研发 SOP (Visual-Driven SOP)**
当你拿到一个需求，并且希望首先得到一张设计图时，请按照以下 4 个阶段推进：

### Phase 0: 需求澄清与视觉偏好
这一步至关重要。除了业务逻辑，你和 `gemini` 必须对 **“长成什么样”** 达成共识。
把原始需求发给 `gemini`，ta会问你：
- 设计风格：是像 Element Plus 那样的标准中后台风，还是像 Ant Design 那样的商务风？
- 核心数据展示：最希望在这个界面上“一眼看到”什么数据？
- 布局偏好：更喜欢上下结构，还是左右结构（左侧是导航/筛选，右侧是内容）？

### Phase 1: 设计图生成与视觉确认 (设计图生成模式 - 核心)
需求清晰后，你正式要求 `gemini` 生成设计图。
将需求描述放到 `../gemini-ui-prompt-template.md` 中，然后把文件内容发给 `gemini`。

### Phase 1.5: 迭代与定稿
`gemini` 会为你生成一张图片。你看了图后，可以随时提出微调意见：
- 你: “把中间那个卡片的背景色改成浅蓝色。”
- 你: “把表格里的‘操作人’列移到最后面。”
- `gemini` 会重新生成图片，直到你满意并确认“以此为准”。

### Phase 2: Trae 的“视觉还原”执行
拿着定稿的这张设计图，你回到 Trae。这一步是 VDD 流程中最令人惊叹的地方：
Trae 具备处理图片的能力。你只需将图片发给它，并下达指令：

**“读取这张设计图，它是我们定稿的 UI。请运用你的 OCR 和 UI 识别能力，将它‘切图’还原为 Vue3 组件代码。要求使用 Element Plus（或你指定的组件库）进行还原，完美复现图片中的布局和样式。”**

1. **OCR 识别文本**：Trae 会识别出图片里的“新增跟进”、“操作人”、“跟进时间”等文本。

2. **布局推理**：它会推断出这是一个 Flex 上下结构，下方是一个 Grid 布局。

3. **组件生成**：它会自动生成比如 FollowupListView.vue、FollowupFilterBar.vue 等文件。

### Phase 3: 绑定数据与全栈整合 (如果有)
当 Trae 帮你完成了静态 UI 的切图后，你再说:

**“现在 UI 已经就绪。请读取我们最开始的【原始业务需求】。根据你识别出的前端 UI 字段，去修改后端 Prisma Schema（确保包含 creator 等审计字段），定义共享 TS 类型，并实现 Express 后端的 Controller 逻辑，最后完成前后端的联调。”**
