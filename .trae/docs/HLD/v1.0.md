# 莱茵酒馆 (Rhine Tavern) - 个人健身与健康管理系统 概要设计文档 (V1.0)

## 1. 系统总体架构与技术栈选型

本项目采用“微信小程序 + Web 后台管理系统”的双端架构，旨在实现移动端的便捷录入与 PC 端的深度数据管理及可视化。

### 1.1 工程化与开发环境
* **代码仓库:** 采用 Monorepo 架构，使用 **Turborepo** 进行统一构建与依赖管理，实现前端、小程序与后端服务之间的类型定义（TS Interfaces）和核心业务逻辑复用。
* **开发工具:** 推荐使用 **Trae** 作为主力 IDE 进行全栈代码编写与调试。

### 1.2 技术栈方案
* **后端 API 服务:** **Node.js** 搭配轻量级框架（如 Express 或 NestJS），提供高并发与灵活的接口服务。
* **数据库:** **PostgreSQL**。利用其强大的关系型处理能力及对 JSONB 的原生支持，完美契合复杂健身计划和灵活的身体指标数据存储。
* **后台管理端:** **Vue 3** + **Vite**，搭配主流 UI 组件库（如 Element Plus），实现高效的 CRUD 和 Echarts 数据看板开发。
* **微信小程序端:** 原生微信小程序开发，或使用基于 Vue 3 的跨端框架（如 Uni-app）以最大化复用技术栈。

## 2. 核心功能模块拆解

系统核心业务逻辑拆分为四大模块：

### 2.1 基础数据源管理模块 (Data Dictionary)
*主要在后台管理系统维护，为小程序端提供标准化的基础数据。*
* **食物库:** 记录常见食物的宏量营养素（碳水、蛋白质、脂肪）及单位热量（kcal/100g）。
* **动作库:** 记录各项健身动作，按目标肌群（胸、背、腿、肩、臂、核心）分类，支持附加动作要领或视频链接。

### 2.2 身体指标追踪模块 (Body Metrics)
*双端协同，小程序主打敏捷录入，后台主打趋势分析。*
* **核心体征:** 每日体重、体脂率打卡。
* **维度数据:** 颈围、胸围、臂围、腰围、臀围、大腿围等周期性记录。
* **状态日志:** 睡眠质量、精神状态、静息心率等辅助恢复指标。

### 2.3 饮食与热量管理模块 (Diet & Nutrition)
* **目标设定:** 基于当前体重设定 TDEE（每日总消耗）及宏量营养素配比目标。
* **饮食日志:** 按餐次（早、中、晚、加餐）快捷录入食物及克数，系统自动计算摄入热量及三大营养素比例。
* **热量监控:** 实时计算当日热量盈亏（卡路里缺口/盈余）。

### 2.4 健身计划与训练日志模块 (Workout Logs)
* **计划定制:** 在后台配置标准训练计划模板（如 PPL 推拉腿分化、五分化等），自由组合动作库内容。
* **训练打卡:** 在小程序端选择当日计划，记录每个动作的“组数”、“重量”和“次数”（Sets × Reps × Weight）。
* **容量统计:** 自动计算单次训练的总容量（Total Volume）及目标肌群负荷。

## 3. 终端功能边界划分

| 终端 | 核心定位 | 主要功能列表 |
| :--- | :--- | :--- |
| **微信小程序** | 贴身助理，强调**轻量、敏捷录入**。 | 1. 快捷添加当日各餐次饮食。<br>2. 健身房内边练边记（组数、重量打卡）。<br>3. 晨起快速记录体重与体脂。<br>4. 查看今日热量剩余与简易周报看板。 |
| **后台管理端** | 控制中心，强调**数据维护、深度可视化**。 | 1. 食物库、动作库的增删改查。<br>2. 训练计划模板的拖拽设计与编排。<br>3. 核心指标（体重/体脂/三大项成绩）的长期折线图分析与报表导出。 |

## 4. 迭代演进计划 (Roadmap)

### Phase 1: MVP（最小可行性产品）- 核心闭环
* 搭建 Turborepo 基础架构，完成 PostgreSQL 数据库表结构设计。
* 实现基础的食物库和动作库管理（后台）。
* 实现小程序端日常体重录入、简易饮食记录和基础的训练流水账功能。

### Phase 2: 数据强化与模板关联
* 引入训练计划模板机制，小程序端支持按模板进行结构化打卡，并计算训练总容量。
* 完善宏量营养素的精确计算与动态热量缺口展示。
* 后台增加复杂视图，实现体重变化与训练容量的趋势可视化分析。

### Phase 3: 智能化与体验升级（远期展望）
* 探索接入 OCR 或 AI 视觉识别 API，实现拍照识别食物并估算热量。
* 增加个人数据导出功能，支持生成专属的年度/季度健身数据报告。
